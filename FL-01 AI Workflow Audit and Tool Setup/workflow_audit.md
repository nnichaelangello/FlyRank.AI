# FL-01: AI Workflow Audit

## 1. Recurring Weekly Tasks & Classification

| Task | Category | Rationale |
| :--- | :--- | :--- |
| 1. Writing Python code for ML/Federated Learning experiments | Collaborate with AI | AI is great for scaffolding boilerplate code, but architectural logic and algorithmic constraints require my direct oversight. |
| 2. Reading and summarizing Scopus-indexed research papers | Delegate to AI with review | AI can extract key methodologies quickly, but I must verify the mathematical claims and experimental setups manually. |
| 3. Data preprocessing and Exploratory Data Analysis (EDA) | Collaborate with AI | I use AI to write complex Pandas/NumPy operations, but the interpretation of data distributions is strictly human. |
| 4. Grading and reviewing student practicum code (Asisten Lab) | Just me | Assessing a student's logical process and giving empathetic, personalized feedback cannot be automated. |
| 5. Drafting sections of academic papers (e.g., Literature Review) | Delegate to AI with review | I provide the core papers and arguments; AI drafts the prose, which I then rigorously edit for academic tone. |
| 6. Building responsive frontend components with Next.js/Tailwind | Fully automate | AI excels at generating standard UI components and CSS classes perfectly, saving hours of manual typing. |
| 7. Designing software architecture for AI deployments (e.g., FastAPI) | Just me | System design requires deep context about server constraints and project requirements that AI doesn't possess. |
| 8. Creating presentation slides for hackathons/competitions | Collaborate with AI | AI helps structure the storytelling and pitch flow, while I design the visuals and specific talking points. |
| 9. Formulating mathematical models for research | Just me | Mathematical novelties and algorithm proofs require deep, focused human cognition without AI hallucination risks. |
| 10. Documenting APIs and writing project READMEs | Fully automate | If the code is well-written, AI can generate comprehensive and accurate documentation almost instantly. |

## 2. Target Tasks for FL-02 to FL-04

1. **Writing Python code for ML/Federated Learning experiments**
   - *What "done well" means:* The code runs without syntax errors, follows best practices for memory management (crucial for CV/FL), and handles edge cases efficiently.
2. **Drafting sections of academic papers (Literature Review)**
   - *What "done well" means:* The text accurately synthesizes the findings of the provided papers without hallucinating citations, maintaining an objective, formal academic tone.
3. **Data preprocessing and Exploratory Data Analysis (EDA)**
   - *What "done well" means:* The generated code successfully cleans the target dataset and outputs clear, interpretable visualizations of the data distribution and missing values.
